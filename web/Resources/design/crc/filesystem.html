FileSystem CRC

Name:

 FileSystem

Comment:

 Provides the functionality of a normal file system i.e. takes operations specified in logical file operations (open file, read bytes from a file, open a directory etc) and translates them into real disk operations. Includes probably providing a directory hierarchy and allocation of free disk blocks.

Implementation is in the form of a class based on MessageHandler.

Collaborators:

 MessageHandler - base class
File - operating system class used to store information about a file
FileAttributes - probably part of previous class stores information about various file attributes length, protections, owner, type etc
Directory
DiskScheduler - "device driver" that fulfills the file systems requests (possibly need multiple DiskSchedulers if FileSystem span many devices)

Responsibilities:

  • FileSystem( DiskScheduler theDiskScheduler )
    - associate FileSystem with a particular DiskScheduler, register with PostOffice

  • AddDiskScheduler( DiskScheduler theDiskScheduler )
    - add a DiskScheduler to the FileSystem

  • various FileSystem functions (following is RCOS list)
    IsEof, Open, Close, Delete, Creat, CloseAll??, Read, Write, Stat, FindFirst, FindNext, Allocate??,
    Additions - ChangeAttributes, GetAttributes, Seek, OpenDir
